# bili2txt-agent 架构设计

## 系统概述

bili2txt-agent 是一个基于飞书长连接的智能机器人，用于自动化处理B站视频内容生成结构化文档。

## 核心模块

### 1. 飞书集成层 (Feishu Integration Layer)

#### feishu_ws_client.py
- **职责**: 维护WebSocket连接，处理心跳和重连
- **关键功能**:
  - WebSocket 连接管理
  - 事件接收和分发
  - 自动重连机制
  - 心跳保持

#### feishu_handler.py
- **职责**: 处理飞书消息事件
- **关键功能**:
  - 消息接收和解析
  - 发送消息回复
  - 用户ID管理

### 2. 业务处理层 (Business Logic Layer)

#### bilibili_downloader.py / bilibili_utils.py
- **职责**: 下载B站视频音频
- **策略**:
  - 优先使用 yt-dlp（音频下载，1-2秒）
  - 回退到 you-get（完整视频，5-10分钟）
- **关键功能**:
  - 视频链接解析
  - 音频流提取
  - 下载进度管理

#### audio_utils.py
- **职责**: 音频处理
- **关键功能**:
  - 音频提取（使用FFmpeg）
  - 格式转换
  - 质量优化

#### asr_utils.py
- **职责**: 语音识别
- **关键功能**:
  - Whisper模型加载
  - 音频转文字
  - 语言检测

#### llm_utils.py
- **职责**: AI文本处理
- **关键功能**:
  - 原文精转（DeepSeek API）
  - 关键纪要生成
  - Prompt优化

#### doc_utils.py
- **职责**: 飞书文档创建
- **关键功能**:
  - Block-based API调用
  - Markdown格式渲染
  - 文档权限管理

### 3. 工具层 (Utility Layer)

#### config.py
- **职责**: 配置管理
- **关键功能**:
  - 环境变量加载
  - 配置验证
  - 默认值管理

#### utils.py
- **职责**: 通用工具函数
- **关键功能**:
  - 文件清理
  - 日志配置
  - 辅助函数

### 4. 协调层 (Coordination Layer)

#### task.py
- **职责**: 任务流程编排
- **关键功能**:
  - 完整处理流程
  - 异步任务调度
  - 错误处理和重试
  - 进度通知

## 数据流

### 正向数据流

```
用户消息 → WebSocket → 事件处理
    ↓
视频ID提取 → 视频下载 → 音频提取
    ↓
语音识别 → 文本精转 → 摘要生成
    ↓
文档创建 → 权限设置 → 链接生成
    ↓
用户通知 ← WebSocket ← 链接返回
```

### 反向数据流

```
错误/异常 → 错误处理 → 用户通知
    ↓
日志记录 → 存储缓存
```

## 关键设计决策

### 1. WebSocket 长连接模式
**优势**:
- 无需公网IP
- 实时双向通信
- 适合本地部署

**实现**:
- 自动重连机制
- 心跳保活
- 事件订阅模式

### 2. 音频优先下载策略
**优势**:
- 300倍速度提升（1-2秒 vs 5-10分钟）
- 99.8%空间节省（1MB vs 500MB）
- 降低带宽需求

**实现**:
- yt-dlp优先，you-get备用
- 音频流提取
- 自动回退机制

### 3. 异步任务处理
**优势**:
- 不阻塞WebSocket连接
- 支持并发处理
- 更好的用户体验

**实现**:
- Python asyncio
- 后台任务调度
- 进度通知

### 4. 本地缓存策略
**优势**:
- 防止数据丢失
- 支持离线查看
- 便于调试

**实现**:
- Markdown文件缓存
- 结构化存储
- 自动清理
