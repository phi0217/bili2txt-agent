bili2txt-agent 开发提示词（飞书长连接模式）
请根据以下要求，生成一个完整的 Python 项目代码，项目名称为 bili2txt-agent，使用 Python 3.11。

项目概述
开发一个飞书机器人，用户通过飞书发送 B 站视频链接（如 https://www.bilibili.com/video/BV1xx...）或 AV/BV 号（如 av123456 或 BV1xx...），机器人自动完成以下操作：

使用正则表达式提取视频 ID（只取第一个，无匹配则返回错误）；

下载视频；

提取音频；

使用本地 Whisper 模型将音频转为文字；

调用 DeepSeek API 对原始文字进行精转，生成可读性强的文章，要求在开头添加“【摘要】”后跟简短摘要，换行后添加“【原文整理】”后跟精转后的全文；

将精转结果上传至飞书云文档，获取公开分享链接；

将文档链接通过飞书消息返回给用户。

通信方式说明
采用飞书官方推荐的长连接模式（WebSocket）接收事件，无需公网 IP 和 HTTP 回调配置。

机器人程序启动后，会与飞书服务器建立持久 WebSocket 连接，飞书将用户消息实时推送过来。

这种方式更简单、安全，适合本地部署或内网环境。

功能需求
1. 飞书长连接客户端
使用飞书 Python SDK（lark-oapi）的 WebSocket 客户端功能。

初始化客户端时需提供 App ID 和 App Secret（从配置读取）。

设置事件处理器，重点处理 im.message.receive_v1 事件。

在事件回调中解析消息内容（需解密）和发送者 open_id。

若消息中未匹配到视频 ID，立即调用飞书 API 发送错误提示“无法识别视频ID，请发送正确的B站链接或AV/BV号”。

若匹配到视频 ID，则启动异步后台任务处理视频（注意不要阻塞事件循环）。

客户端需保持运行，自动重连，处理心跳。

2. 视频 ID 解析
仅使用正则表达式，不调用任何 LLM 兜底。

正则规则：

BV 号：BV[a-zA-Z0-9]+

AV 号：av\d+（忽略大小写，输出小写）

取第一个匹配项，无匹配返回 None。

3. 后台任务处理
后台任务函数接收 video_id 和 user_id，顺序执行以下步骤，每一步都要捕获异常，失败则通过飞书告知用户错误，并清理已产生的临时文件：

下载视频：调用 you-get 命令行工具下载视频到临时目录，返回视频文件路径。

提取音频：调用 ffmpeg 从视频中提取 MP3 音频，返回音频文件路径。

语音识别：使用本地 Whisper 模型（如 tiny）将音频转为文字，返回原始文本。

LLM 精转：调用 DeepSeek API，按要求的格式生成精转文本，返回包含“【摘要】”和“【原文整理】”的文本。

上传云文档：使用飞书云文档 API 创建文档，将精转文本写入文档，设置公开访问权限，返回文档分享链接（形如 https://www.feishu.cn/docx/{doc_id}）。

发送结果：通过飞书 API 将文档链接发送给用户。

清理文件：删除临时视频、音频文件。

4. 视频下载
使用 you-get 命令：you-get -o {temp_dir} {video_url}。

视频 URL 构造：https://www.bilibili.com/video/{video_id}。

下载完成后，从临时目录中找到最新创建的视频文件（按创建时间排序），返回绝对路径。

设置超时（如 300 秒），超时或命令失败则抛出异常。

5. 音频提取
使用 ffmpeg 命令：ffmpeg -i {video_path} -q:a 0 -map a {audio_path} -y。

音频路径与视频同目录，扩展名改为 .mp3。

超时 300 秒，失败抛异常。

6. 语音识别
使用 openai-whisper 库，加载模型（推荐 tiny，可配置）。

在模块级别加载一次模型，避免重复加载。

调用 model.transcribe(audio_path, language='zh')，返回 result["text"]。

7. LLM 精转
使用 openai 库，配置 base_url 为 DeepSeek API 地址，api_key 从环境变量读取。

构造提示词（prompt）如下：

text
请将以下语音转写的文本整理成一篇可读性强的文章，修正错别字和不通顺之处，使其符合书面语表达。然后在文章开头用“【摘要】”开头添加一个简短的摘要，概括核心内容。换行后，用“【原文整理】”开头输出精转后的全文。原文如下：
{original_text}
调用 deepseek-chat 模型，设置 max_tokens 适当（如 3000），返回结果。

8. 飞书云文档创建与分享
使用飞书 Python SDK（lark-oapi）操作。

步骤：

获取 tenant_access_token（SDK 自动管理）。
创建空文档：POST /open-apis/docx/v1/documents，获得 document_id。
向文档添加内容：使用 PATCH /open-apis/docx/v1/documents/{document_id}/blocks/{block_id}/children 或批量更新子块接口，将精转文本作为文本块添加。简化：将整个文本作为一个块添加。
设置文档公开访问：POST /open-apis/drive/v1/permissions/{document_id}/public，设置 external_access=True。
构造分享链接：{FEISHU_DOMAIN}/docx/{document_id}，其中 FEISHU_DOMAIN 为配置的飞书域名（如 https://www.feishu.cn）。
若创建文档或写入失败，抛出异常。

9. 飞书消息发送
使用飞书 SDK 发送文本消息。

提供两个函数：send_message(user_id, text) 和 send_document_link(user_id, doc_url)（后者直接调用前者发送链接文本）。

10. 错误处理与日志
每个步骤均需 try-except，捕获异常后记录日志（使用 logging），并通过飞书发送错误提示给用户。

清理函数应能删除文件或目录，忽略不存在的文件。

技术栈
Python 3.11

lark-oapi（飞书官方 SDK，支持 WebSocket 客户端）

openai（用于调用 DeepSeek API）

openai-whisper（本地语音识别）

ffmpeg-python 或直接 subprocess 调用 ffmpeg（需预装 FFmpeg）

you-get（需预装，通过 subprocess 调用）

其他：re, os, subprocess, time, json, asyncio, threading 等标准库。

部署说明
环境准备：

安装 Python 3.11。

安装 FFmpeg 并加入 PATH。

安装 you-get：pip install you-get。

项目部署：

克隆代码，创建虚拟环境，安装依赖：pip install -r requirements.txt。

配置 .env 文件。

运行：python main.py（程序将持续运行，保持长连接）。

飞书应用配置：

在飞书开放平台创建应用，开启机器人能力。

在“事件与回调”配置中，选择“使用长连接接收事件”。

订阅 im.message.receive_v1 事件。

授予应用权限：接收消息、发送消息、创建文档、管理文档。

获取 App ID, App Secret，填入 .env。

注意事项
所有临时文件必须在使用后删除，避免磁盘占用。

Whisper 模型首次运行会自动下载，可提前下载好放到缓存目录。

DeepSeek API 调用注意 token 限制和费用。

后台任务可能耗时较长，需使用异步或线程避免阻塞事件循环。

飞书长连接客户端会自动重连，无需额外处理。
